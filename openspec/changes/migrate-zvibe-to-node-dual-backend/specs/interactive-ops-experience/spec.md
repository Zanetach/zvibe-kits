## ADDED Requirements

### 需求:中文交互输出
系统必须默认输出中文交互文案，覆盖 setup、config、status 及启动流程的关键提示。

#### 场景:默认中文文案
- **当** 用户执行任一核心命令
- **那么** 系统必须以中文输出关键状态、错误原因与建议动作

### 需求:setup 可重复修复
系统必须支持可重复执行的 setup 修复流程，且禁止重复安装已满足依赖。

#### 场景:setup repair
- **当** 用户执行 setup 且部分依赖已满足、部分依赖缺失
- **那么** 系统必须跳过已满足项并仅修复失败项

### 需求:status 分层诊断
系统必须在 status 中分层展示环境层、配置层、运行层状态。

#### 场景:分层状态展示
- **当** 用户执行 `zvibe status`
- **那么** 系统必须输出环境检查、配置合法性和运行可用性三类结果

### 需求:可行动错误提示
系统必须将底层错误映射为可行动提示，并包含建议命令或修复路径。

#### 场景:后端不可用错误映射
- **当** 后端启动失败
- **那么** 系统必须输出标准错误码与明确修复建议，而不只输出底层异常文本
